# 产品需求文档 (PRD)：ink-xY 智能小说创作工坊

## 1. 产品概述与愿景

### 1.1 产品定位
**ink-xY** 是一款专为网文作家、独立编剧及严肃文学创作者打造的 **集“非线性大纲、真理设定提取、自主章节写作与章节版本回滚”于一体的专业级智能小说创作工坊 (Novel IDE)**。

### 1.2 核心痛点与愿景
* **痛点**：
  * 传统 AI 写作软件（基于普通对话框）由于上下文窗口限制，写长篇时极易发生**“人设崩塌”**和**“剧情前后矛盾”**。
  * 小说大纲和分支剧情的探索十分繁琐，传统线性文档极难管理复杂的非线性故事走向（A/B 结局）。
  * 网页版 AI 助手无法直接读写本地的章节草稿与人设文件，人机协同存在断层。
* **愿景**：
  将 `ink-xY Novel Studio` 精美实用的**双栏桌面编辑器、Markdown 预览器和会话分支树**，与 `InkOS` 领先的**小说自主写作/审计/快照引擎**深度整合，打造一款极速、私密、强大且富有灵感的本地桌面端协同写作环境，实现“人机无缝共创”。

---

## 2. 目标用户画像

* **网络小说作家**：日更需求高，极其依赖灵感发散、章节扩写与剧情防崩。
* **游戏文案与编剧**：设计海量支线任务、NPC 对话树以及错综复杂的非线性故事网。
* **严肃文学/同人创作者**：注重文字质感，对人设前后连贯性、背景世界观设定（Lore）有极高一致性要求。

---

## 3. 系统架构设计

**ink-xY** 将采用轻量化的 **“静态前端 + 系统 Node 壳 + Inkos CLI”** 架构模型：

```
+-------------------------------------------------------------+
|                     ink-xY Desktop App                      |
|                                                             |
|   +--------------------------+  +------------------------+  |
|   |         左栏/中栏        |  |          右栏          |  |
|   |   小说大纲、人设、章节草稿   |  |   AI 写作助手、审计报告 |  |
|   |      (Markdown 编辑器)    |  |     (交互会话窗口)     |  |
|   +------------+-------------+  +-----------+------------+  |
|                |                            |               |
+----------------|----------------------------|---------------+
                 v                            v
      [REST API & 文件系统读写]        [EventSource 监听]
                 |                            |
                 v                            v
  +-----------------------------------------------------------+
  |              Next.js In-Process Server (3040)             |
  |                                                           |
  |   +---------------------------------------------------+   |
  |   |           lib/npx.ts (系统寻址进程包装器)           |   |
  |   |   直接调度: npx inkos [init | write | audit]       |   |
  |   +---------------------------------------------------+   |
  +-----------------------------------------------------------+
                                |
                                v
               [硬盘工作区: 我的长篇小说/ (Novel CWD)]

```

## 4. 核心功能需求 (Functional Requirements)

### 4.1 工作区与大纲管理 (Workspace & FileExplorer)
* **需求**：提供标准的小说项目骨架，自动管理写作资产。
* **细节**：
  * 一键初始化小说工作区，自动在本地创建：`故事大纲.md`、`角色设定/`、`世界观设定/`、`章节草稿/` 目录。
  * 左侧内置轻量级 `FileExplorer`，支持随时创建、重命名、编辑不同的章节和人设卡片。
  * **自动防护**：文件写入 `Temp/` 或草稿箱时，系统自动生成包含 `*` 的 `.gitignore`，防止创作草稿被意外 Git 提交。

### 4.2 真理提取与人设防崩系统 (Truth System)
* **需求**：基于 `InkOS` 后端，自动从已写的章节中反向提炼设定，并确保写作时逻辑不冲突。
* **细节**：
  * **真理提取 (Reverse Engineering)**：一键扫描指定文件夹下的旧章节，由智能体分析并自动在 `角色设定/` 和 `世界观设定/` 中补充细节（如主角的武器、性格变化、已发生的事件）。
  * **人设保护机制**：AI 助手在右侧扩写章节时，强制静默读取 `角色设定/` 文件夹下的 markdown 描述作为核心上下文，一旦出现逻辑冲突（例如已死角色复活、主角性格突变）立即警告。

### 4.3 故事树与剧情分叉 (Non-linear Branching & Snapshot)
* **需求**：作者能够视觉化推演非线性剧情，并一键回滚章节。
* **细节**：
  * **分叉写作 (Forking)**：写到关键剧情分水岭时，作者可在左侧会话树中点击 **"Fork"**，分出“走向 A”和“走向 B”两条独立故事线分别推进。
  * **版本快照 (Snapshot)**：对接 `InkOS` 的快照机制，记录每一章在 AI 辅助下的每一次重大改动，允许一键在界面上回滚至 3 次修改前的版本。

### 4.4 智能人机协同写作姬 (Human-in-the-Loop Writing Copilot)
* **需求**：彻底摒弃繁杂的代码执行工具，聚焦于纯粹的文学文本辅助。
* **细节**：
  * **工具极简化**：完全关闭底层的终端、Skills 安装、代码执行面板。AI 助手进入纯文本生成模式，响应速度相比代码智能体提升 5 倍以上。
  * **多角色 Gems 助手**：内置 4 大预设 Gems：
    * `大纲策划师`：辅导宏观起承转合与冲突设置。
    * `角色塑造师`：填充人物性格，润色对话口吻。
    * `细节扩写师`：划词扩写，把动作、环境描写打磨得极具文学美感。
    * `纠错校对师`：自动扫描章节中的错别字与语病。

## 5. 交互与视觉设计需求 (UX/UI)

* **禅意护眼换肤 (Zen Mode Theme)**：
  * 提供米黄色（仿纸张）、浅绿色（护眼）以及极简深灰色（无干扰）三种配色主题。
  * 去除所有极客程序员元素（如终端黑框、API 键配置区），仅保留精美的双栏编辑器。
* **文学质感字体**：
  * 前端默认采用高雅的衬线字体（Serif，如宋体、楷体、Lora、Outfit），显著降低长时间阅读的眼部疲劳。
* **防抖渲染保护 (Memory Defense)**：
  * 完美集成 `1500ms` 本地大文件防抖，防止在 AI 写入多达几万字的大纲或章节时，前端由于频繁 Canvas/Markdown 重绘而产生内存溢出和卡顿。

---

## 6. 非功能性需求

* **运行环境**：独立桌面客户端（Windows `.exe` / macOS `.dmg`），无需用户安装 Node.js 或搭建任何环境即可直接双击运行。
* **绝对私密性**：所有的写作草稿、大纲、人物卡片等核心资产**全部加密保存在用户本地磁盘**上，大语言模型仅作为生成工具使用，绝不上传任何创作隐私。
