## 1. 快速开始

### 1.1 环境要求

- **Node.js** >= 18（推荐 LTS）
- **pi 编码智能体** 已安装并可正常运行（[GitHub](https://github.com/badlogic/pi-mono)）
- 浏览器推荐 Chrome / Edge / Firefox 最新版本

### 1.2 安装与启动

**方式一：零安装（推荐）**

`ash
npx @zwbigi/pi-agent-xy@latest
`

**方式二：全局安装**

`ash
npm install -g @zwbigi/pi-agent-xy
pi-agent-xy
`

启动后自动打开浏览器访问 [http://localhost:30142](http://localhost:30142)。

### 1.3 自定义启动参数

`ash
# 指定端口
pi-agent-xy --port 8080
pi-agent-xy -p 8080

# 指定主机名（仅本机访问）
pi-agent-xy --hostname 127.0.0.1
pi-agent-xy -H 127.0.0.1

# 组合使用
pi-agent-xy -p 8080 -H 127.0.0.1

# 通过环境变量
PORT=8080 pi-agent-xy
`

### 1.4 开发模式（贡献者）

`ash
git clone <repo-url>
cd pi-web-src
npm install
npm run dev（或者：node bin\pi-web.js）     # 端口 30142，支持热重载    
`

> **注意**：开发时请勿运行 
ext build，会污染 .next/ 目录导致 
pm run dev 异常。

## 2. 界面总览

Pi Agent xY 采用三栏布局：

`
┌──────────────┬──────────────────────────────────────────┐
│              │  顶部栏（分支导航 · 系统提示 · 会话统计）│
│   侧边栏     ├──────────────────────────────────────────│
│              │  Tab 栏（Chat · 文件标签…）               │
│  · 会话列表  ├──────────────────────────────────────────│
│  · 文件浏览  │                                          │
│  · Gem 面板  │        聊天消息区                        │
│  · 设置入口  │     （ChatWindow / FileViewer）           │
│              │                                          │
│              ├──────────────────────────────────────────│
│              │  输入栏（模型·思考·工具·压缩·发送）       │
└──────────────┴──────────────────────────────────────────┘
`

### 2.1 侧边栏

侧边栏包含以下区域：

- **会话列表**：按工作目录分组，展示所有历史会话
- **文件浏览器**：浏览当前工作目录的文件树
- **Gem-xY 面板**：管理自定义智能体模板
- **底部工具栏**：Models 配置入口、Skills 管理入口

点击侧边栏顶部的折叠按钮可收起/展开侧边栏。

### 2.2 顶部栏

顶部栏显示当前会话的：

- **分支导航器**（BranchNavigator）：可视化展示会话内分支树
- **系统提示词**：查看/编辑当前会话的系统提示词
- **会话统计**：Token 用量（input / output / cache read / cache write）和费用
- **上下文用量**：上下文窗口使用百分比

### 2.3 Tab 栏

- **Chat 标签**：默认标签，展示聊天界面
- **文件标签**：从文件浏览器打开文件时，在新标签页中展示内容

## 3. 会话管理

### 3.1 浏览会话

会话列表按 **工作目录 (cwd)** 分组展示，每个会话显示：

- 会话名称（用户自定义或自动生成）
- 创建时间 / 最后修改时间（相对时间，如"3h ago"）
- 消息数量
- 首条消息预览

> **孤儿会话**：如果会话文件首行无法正确解析，会显示「不完整」徽章，不可点击进入。

### 3.2 创建新会话

1. 点击侧边栏顶部的 **+** 按钮
2. 选择 **工作目录 (cwd)** — 最近使用的 5 个目录会优先展示
3. （可选）选择 **Gem 模板** — 使用预设的自定义智能体
4. 配置 **模型**、**工具预设**、**思考级别**
5. 输入首条消息并发送

### 3.3 打开会话

在会话列表中点击任意会话即可加载其历史消息。支持通过 URL 参数直接定位到特定会话。

### 3.4 删除会话

在会话列表中右键或使用会话项上的删除按钮。删除会话会：

- 删除对应的 .jsonl 文件
- 自动更新所有子会话（Fork 产生的）的父级引用

### 3.5 重命名会话

在会话列表中可对会话进行重命名，名称会保存到 .jsonl 文件的 session_info 条目中。

## 4. 与智能体对话

### 4.1 发送消息

在底部输入栏中输入文字，按 **Enter** 发送（**Shift + Enter** 换行）。

#### 附加图片

点击输入栏左侧的 **图片按钮** 或直接 **拖放图片** 到输入框，可附加图片作为上下文发送给智能体。支持的格式：PNG、JPEG、WebP 等。

### 4.2 流式输出

消息发送后，智能体的回复以 **SSE（Server-Sent Events）** 实时推送，你可以看到：

- **文字逐字输出** — 助手回复实时渲染
- **思维链 (Thinking)** — 折叠展示智能体的推理过程，点击展开查看详情
- **工具调用 (Tool Call)** — 展示智能体正在使用的工具名称和输入参数
- **工具结果 (Tool Result)** — 展示工具执行的输出结果，区分成功/错误状态
- **图片** — 助手回复中的图片内联展示

### 4.3 中断智能体 (Interrupt)

当智能体正在运行时，输入栏会切换为 **中断模式**：

- 点击 **Stop** 按钮或按快捷键打断当前运行
- 你也可以在中断前输入文字 — 这会作为"引导"消息在中断后发送

### 4.4 追加消息 (Follow-up)

当智能体完成回复后，你可以直接输入新消息追加对话，无需任何额外操作。

### 4.5 Fork（分叉会话）

如果你想从某个对话节点"另起炉灶"：

1. 将鼠标悬停在任意 **用户消息** 上
2. 点击消息右侧的 **Fork** 按钮
3. 系统会创建一个 **全新的独立会话文件**，从该消息开始
4. 新会话在侧边栏中显示为原会话的子节点

> **Fork vs 分支**：Fork 创建独立文件，适合长期分叉探索；分支（见下节）在同一文件内切换，适合临时回退。

### 4.6 会话内分支 (In-session Branch)

pi 智能体支持在同一个会话文件内创建多个分支：

- **分支导航器**：顶部栏的 BranchNavigator 展示当前会话的树形分支结构
- **切换分支**：点击分支树中的节点即可切换到该分支
- **创建分支**：使用"Continue"按钮从当前叶子节点创建新分支

### 4.7 压缩会话 (Compact)

对于较长的会话，上下文窗口可能不够用：

1. 点击输入栏左侧的 **Compact** 按钮
2. 智能体会对历史消息进行摘要压缩
3. 压缩过程中按钮显示为禁用状态
4. 压缩完成后继续对话，上下文窗口得到释放

### 4.8 会话统计

顶部栏实时显示当前会话的统计信息：

| 指标 | 说明 |
|------|------|
| Input Tokens | 输入 token 数 |
| Output Tokens | 输出 token 数 |
| Cache Read | 缓存读取 token 数 |
| Cache Write | 缓存写入 token 数 |
| Cost | 本次会话累计费用 |
| Context Usage | 上下文窗口使用百分比 |

## 5. 模型与工具配置

### 5.1 模型选择

#### 新建会话时选择

在创建新会话时，从模型下拉列表中选择 Provider + Model。

#### 对话中切换

在输入栏左侧点击当前模型名称，弹出模型列表，选择新模型即可切换。切换后后续消息使用新模型，切换记录会保存到会话文件中。

#### 模型配置

点击侧边栏底部的 **Models** 按钮打开模型配置面板：

- 查看当前可用模型列表
- 编辑 models.json 配置文件（位于 ~/.ink/agent/models.json）
- 设置默认模型（位于 ~/.ink/agent/settings.json）

### 5.2 工具预设

工具控制智能体可以使用的"能力"。在输入栏左侧点击工具图标选择预设：

| 预设 | 包含工具 | 适用场景 |
|------|----------|----------|
| **Off** | 无 | 纯对话、翻译、写作等不需要操作文件的场景 |
| **Low** (Default) | read · bash · edit · write | 日常编码、代码修改 |
| **High** (Full) | read · bash · edit · write · grep · find · ls | 复杂项目探索、大规模重构 |

### 5.3 思考级别 (Thinking Level)

在输入栏左侧点击思考图标选择级别：

| 级别 | 说明 |
|------|------|
| **auto** | 沿用 pi 默认设置 |
| **off** | 关闭推理 |
| **minimal** | 最少推理 |
| **low** | 低强度推理 |
| **medium** | 中等推理 |
| **high** | 高强度推理 |
| **xhigh** | 最高推理（适合复杂问题，消耗更多 token） |

## 6. Gem-xY 自定义智能体

Gem-xY 让你创建个性化的智能体模板，为不同场景预设专属配置。

### 6.1 什么是 Gem？

一个 Gem 是一个智能体模板，包含：

- **头像 (Emoji)** — 如 🔍、📝、🛡️
- **名称与描述** — 如"代码审查员"、"翻译官"
- **系统提示词** — 定义智能体的角色和行为准则
- **绑定模型** — 指定使用哪个 Provider + Model
- **工具过滤** — 限制智能体可用的工具（如仅允许只读工具）
- **知识库文件** — 附加本地文件作为 RAG 上下文

### 6.2 创建 Gem

1. 点击侧边栏中的 **Gem-xY** 面板
2. 点击 **新建** 按钮
3. 在编辑器中填写：
   - 选择一个 Emoji 作为头像
   - 输入名称和描述
   - 编写系统提示词（定义智能体的角色、行为规范）
   - 选择绑定的模型
   - 勾选允许使用的工具
   - （可选）添加知识库文件
4. 点击 **保存**

### 6.3 使用 Gem 创建会话

1. 点击 **+** 创建新会话
2. 在创建界面中选择一个 Gem 模板
3. 该 Gem 的系统提示词、模型、工具配置会自动应用
4. 正常对话即可 — 智能体会按照 Gem 定义的角色和规则运行

### 6.4 管理 Gem

在 Gem-xY 面板中：

- **编辑**：点击 Gem 卡片进入编辑器修改配置
- **删除**：删除不需要的 Gem 模板
- **复制**：基于现有 Gem 快速创建变体

Gem 数据存储在 ~/.ink/agent/gem_xy.json。

## 10. 常见问题 (FAQ)

### Q: 启动后页面空白或报错？

**A:** 确认以下条件：
- Node.js >= 18
- pi 编码智能体已正确安装
- 端口 30142 未被占用（可通过 --port 指定其他端口）

### Q: 会话列表不显示？

**A:** Pi Agent xY 读取 ~/.ink/agent/sessions 目录下的 .jsonl 文件。如果使用了自定义数据目录，请设置环境变量：

`ash
PI_CODING_AGENT_DIR=/path/to/your/sessions pi-agent-xy
`

### Q: 消息发送后没有响应？

**A:** 可能原因：
- 智能体 AgentSession 未能创建 — 检查 pi 编码智能体是否正常安装
- 模型配置有误 — 在 Models 面板中检查 Provider 和 API Key 配置
- 网络问题 — 确认能正常访问 AI Provider 的 API

### Q: 如何同时运行多个实例？

**A:** 为每个实例指定不同的端口：

`ash
pi-agent-xy -p 30142   # 实例 1
pi-agent-xy -p 30143   # 实例 2
`

### Q: Fork 和分支有什么区别？

**A:**

| | Fork | 分支 (Branch) |
|---|------|--------------|
| 文件 | 创建新的独立 .jsonl 文件 | 在同一文件内切换 |
| 侧边栏 | 显示为子节点 | 通过分支导航器切换 |
| 适用场景 | 长期分叉探索不同方向 | 临时回退尝试不同回复 |

### Q: 压缩 (Compact) 是什么？

**A:** 压缩是对长会话的摘要优化。当上下文窗口即将用尽时，压缩会将早期消息总结为摘要，释放上下文空间，使对话可以继续进行。

### Q: 如何修改默认模型？

**A:** 在侧边栏底部点击 **Models** 按钮，或直接编辑 ~/.ink/agent/settings.json 中的 defaultModel 字段。

### Q: 如何让智能体只读代码不修改？

**A:** 两种方式：
1. 创建会话时将工具预设设为 **Off**
2. 创建一个 Gem，工具过滤中只勾选 
ead、grep、ind、ls

### Q: 页面刷新后对话丢失？

**A:** 不会丢失。所有消息都持久化在 .jsonl 文件中。页面刷新后会自动重新加载会话，如果刷新时智能体正在运行，SSE 连接会自动恢复。

### Q: 如何远程访问？

**A:** 启动时绑定到所有网络接口：

`ash
pi-agent-xy -H 0.0.0.0
`

然后通过 http://<你的IP>:30142 访问。局域网内的设备（如手机、平板）均可访问。

> **安全提示**：远程访问时请确保网络环境安全，Pi Agent xY 目前无内置身份验证。
