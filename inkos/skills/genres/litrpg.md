## 题材禁忌

- 设定好系统规则后肆意改动——读者会仔细核对每一个数值与逻辑。
- 毫无道理的力量暴涨——成长过程必须遵循已确立的系统规则与数值框架。
- “属性面板狂躁症”——每章大段倾倒属性字数，或动辄罗列长达数页的面板。
- 瞬间掌控技能——刚学到一个复杂技能就立即使用得完美无缺。
- 主角开局即无敌——这会剥夺所有的危机感与成长乐趣。
- 在紧张的战斗过程中插入海量、高频的系统播报，打断叙事节奏。
- 女性角色流于套路化——被窄化为单纯的“花瓶女主”或“卧病在床需要拯救的妹妹/女儿”等刻板工具人。

## 系统设计规则

- 属性、技能与系统规则一旦确立，绝对不能前后吃书或相互矛盾。
- 面板呈现应作为高光成就时刻的标记，而不是无意义的日常灌水。
- 衍生属性（如生命值、法力值）必须与基础主属性逻辑挂钩。
- 技能解锁必须有门槛——与风险、牺牲或高难度解谜相绑定。
- 同类型资源吞噬/吸收必须遵循收益递减机制，不得默认始终满额结算。
- System UI (blue boxes) appears after action/revelation, never mid-combat (系统UI应当在行动结束后出现，绝不能在中途激烈打斗时连续插播。)
- 赋予系统一致的人设和语气——可以是绝对冰冷的公事公办、古老沧桑的质感、戏谑活泼的引导音，或临床医学般的理性。

## 叙事节奏指导

- 章节字数黄金比例：结合面板密度，控制在2.8k-3.5k字左右。
- 章节结构：钩子/引子 -> 行动/探索 -> 系统交互/属性收益 -> 悬念尾声。
- 前期章节：保持较高频的升级体验以吸引读者（每1-3章有明确提升）。
- 中期剧情：获取难度的曲线抬升，每5-10章一次突破，同时推高叙事张力。
- 后期剧情：大境界或段位跨越应当是稀有、极其庄重且作为高潮爆发点呈现。
- 节奏验证：第3卷的主角应该能碾压第1卷的自己，但当前的挑战对手绝不应显得平庸无趣。
- 优先在旁白中将升级结果融入具象描写（利于听书体验），再在文末提供详细的数值面板供硬核读者查阅。
